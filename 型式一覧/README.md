# 型式と燃費値一覧

国土交通省の情報を元に平成25年度以降、各年代に発表された型式と燃費値をモード毎にまとめました。  
- WLTCモード    > output_WLTC.csv  
- JC08モード    > output_JC08.csv  
- 10・15モード  > output_10-15.csv  
- JH25モード    > output_JH25.csv  
- JH15モード    > output_JH15.csv  
※モードは計算方式（計算の元となる算出方法）の違い  
※WLTCモードは 2017年（平成29年）から導入   
※JH25モードは 2019年3月29日（平成31年3月29日）に公布・施行  
　https://www.mlit.go.jp/common/001395752.pdf  
※JH15モードは重量車モードとも呼ばれます。  
※平成24年度以前はPDF資料の為、別で対応  

- 最新の情報は https://www.mlit.go.jp/jidosha/jidosha_mn10_000002.html から各年代のファイルをダウンロードしてください。  
  ※環境：自動車燃費一覧 - 国土交通省.url　からショートカットでいけます。  

- 各モードの説明はGoogle Drive 以下URLにて  
  https://docs.google.com/presentation/d/1dAgt6wFna-3PncQdlZaIhdJ0RsqNz8nU/edit?usp=drive_link&ouid=111979128271137581146&rtpof=true&sd=true


# 各outputファイルへの新規追加

各outputファイルへの新規追加を行うには 最新のデータを  
https://www.mlit.go.jp/jidosha/jidosha_mn10_000002.html 各年代にアクセスし  
ファイルをダウンロード後、フォルダーにまとめ ファイル名に **モード名を追記** します。  

Excelファイル統合スクリプト excel_consolidator.py　を実行するとまとめられます。

# Excelファイル統合スクリプト

指定フォルダ内のExcelファイルから特定のデータを抽出し、CSVに出力するPythonスクリプトです。

## 機能

- 指定フォルダ以下の全てのサブフォルダからExcelファイル（.xls / .xlsx）を検索
- ファイル名に特定のキーワード（WLTC、JC08、10・15、JH25）を含むファイルをフィルタリング
- 各Excelファイルの全シートの4～8行目から以下の情報を抽出：
  - 車名
  - 通称名
  - 型式
  - 燃費値（km/L）
- フォルダ名とファイル名も一緒に出力
- キーワードごとに別々のCSVファイルに出力

## 必要な環境

- Python 3.6以上
- 必要なライブラリ：
  - openpyxl（xlsxファイル用）
  - xlrd（xlsファイル用）

## インストール

```bash
pip install -r requirements.txt
```

## 使い方

1. スクリプトと同じディレクトリに `型式一覧` フォルダがあることを確認
2. スクリプトを実行：

```bash
python excel_consolidator.py
```

## 出力ファイル

- `output_WLTC.csv` - ファイル名に「WLTC」を含むファイルからのデータ
- `output_JC08.csv` - ファイル名に「JC08」を含むファイルからのデータ
- `output_10-15.csv` - ファイル名に「10・15」を含むファイルからのデータ
- `output_JH25.csv` - ファイル名に「JH25」を含むファイルからのデータ
- `output_JH15.csv` - ファイル名に「JH15」を含むファイルからのデータ

## カスタマイズ方法

スクリプトの上部にある「設定セクション」で以下をカスタマイズできます：

### 1. 抽出項目の追加

`OUTPUT_HEADERS` リストに新しい項目を追加：

```python
OUTPUT_HEADERS = [
    'フォルダ名',
    'ファイル名',
    '車名',
    '通称名',
    '型式',
    '燃費値（km/L）',
    '新しい項目',  # <- 追加
]
```

### 2. ファイルフィルタの追加

`FILE_FILTERS` 辞書に新しいキーワードと出力ファイル名を追加：

```python
FILE_FILTERS = {
    'WLTC': 'output_WLTC.csv',
    'JC08': 'output_JC08.csv',
    '10・15': 'output_10-15.csv',
    'JH25': 'output_JH25.csv',
    '新キーワード': 'output_new.csv',  # <- 追加
}
```

### 3. データ抽出行の変更

`TARGET_ROWS` で抽出対象の行範囲を変更：

```python
TARGET_ROWS = range(4, 9)  # 4～8行目
```

### 4. 基準フォルダの変更

```python
BASE_FOLDER = './型式一覧'
```

## 注意事項

- **データの自動正規化**
  
  **型式の正規化:**
  - 全角英数字を自動的に半角に変換
  - 全角ハイフン（－、ー）も半角（-）に変換
  - 改行を削除して1行にまとめる
  
  **通称名の正規化:**
  - 記号のみ（※1、※、＊、*2、*、※２、＊２など）の場合は右隣の列から取得
    - 半角記号と全角記号の両方に対応
  - 全角括弧（）を半角()に自動変換
  - 改行を空白に置換して1行にまとめる
  - 連続する空白を1つにまとめる
  - 先頭・末尾の空白を削除
  
  **注釈行の自動除外:**
  - 「（注）」「*印の付いて」「製造事業者」などを含む注釈行を自動除外
  - 「実際に販売されている」「異なる場合があります」などの注意書き行も除外
  - 型式フォーマットチェック：4文字目に「-」がない型式は注釈行として除外
  - データ品質を向上

- **車名の自動補完**
  - 車名が空の場合、Excelのシート名を車名として使用
  - 輸入車などシート名で車名が分かるファイルに対応

- **JH15ファイルの特殊処理**
  - 通称名が型式と同じ場合、結合セルとして扱い前の値を使用
  - 通称名が型式フォーマット（XXX-XXXXXX形式）の場合も同様に処理
  - 型式フォーマットの通称名は記憶せず、前の正しい通称名を維持
  - JH15特有のデータ構造に自動対応

- **空の通称名の自動補完**
  - 通称名が空の場合、前のレコードの通称名を自動的に使用
  - JH15および10・15ファイルで特に有効
  - データの完全性を確保

- **改行を含む記号パターン対応**
  - 『※\n※』のように改行を含む記号パターンも正しく検出
  - スペース区切りの記号パターン（『※ ※』『※1 ※2』など）にも対応
  - 代替列から正しい通称名を取得

- **Excelファイルの結合セルに完全対応**
  - 車名や通称名が結合セルになっている場合、自動的に前の値を引き継ぎます

- **柔軟なヘッダー検出**
  - 「燃費値（km/L）」は改行の有無に関わらず検索されます
    - WLTC/JC08形式：「燃費値（km/L）」
    - JH15/JH25形式：「(km/L)」
  - 通称名の列位置が異なる場合も自動的に検出されます
  - ヘッダーに「車名」「通称名」がないファイルにも対応
    - デフォルトでCol1=車名、Col2またはCol3=通称名として処理

- **ファイルごとに異なる構造に自動対応**
  - ヘッダー行とデータ開始行を自動検出
  - WLTC/JC08: 4～8行目がヘッダー、9行目以降がデータ
  - JH15: 6～7行目がヘッダー、8行目以降がデータ
  - 平成25年3月の一部ファイル: 8行目からデータ開始

- **出力形式**
  - CSVファイルはUTF-8 BOM付きで出力されます（Excelで開く際の文字化け防止）
  - すべてのデータは正規化されて1行ずつ出力されます

- **エラーハンドリング**
  - エラーが発生したファイルはスキップされ、処理は継続されます

## トラブルシューティング

### エラー: `ModuleNotFoundError: No module named 'openpyxl'`

```bash
pip install openpyxl xlrd
```

### 文字化けする場合

CSVファイルはUTF-8 BOM付きで保存されていますが、それでも文字化けする場合は、Excelで開く際に「データ」→「テキストファイル」から開き、エンコーディングを「UTF-8」に指定してください。
